# git関係のコマンド
```
$ git init （リポジトリを作る）
$ git status （状況を確認する）
$ git add （ステージする）
$ git commit （コミットする）
#コミットメッセージは必須なので、メッセージがない場合はコミットがキャンセルされる
$ git log （履歴を見る）
#graphオプションで図を作成、allオプションで全体のコミットを確認可能
$ git branch（ブランチを作る）
# git branch -d branch (branchを削除する)
$ git checkout （チェックアウトする）
$ git merge [ブランチ名] （HEADのあるブランチにマージする）
#git merge --abort(マージを中止する)
#-suquashオプションでそのブランチのコミット全てをまとめたコミットが追加される。
& git revert [打ち消したいコミットID](特定のコミットを打ち消す)
#すでに公開済みである(勝手に削除できない)コミットに対して使用する
$ git rebase(**試したがうまく動かない、もう一回やって無理なら質問**)
#iオプションでコミットの書き換え、入れ替え、削除、統合を行うことができる。
#pushする前にコミットコメントをきれいに書き直す
#意味的に同じ内容のコミットをわかりやすいように1つにまとめる
#pick ~と書かれた行が複数表示されるので、2つ目以降のpickをsquashに変更する(各操作に対応した文字列に変更する)
コミット漏れしたファイルを後から追加する
$ git reset(いらなくなったコミットを捨てる。3つのオプションによって影響を及ぼす範囲を指定する。)
#対応表は章末に示す
#git reset --hard HEAD~のようにHEADからいくつに影響を及ぼすかを指定する
#git reset --hard ORIG_HEADでreset前の状態に戻すことができる
$ git cherry-pick　コミットID(別のブランチから指定したコミットをコピーして、現在のブランチに取り込む)
#間違えて追加したコミットを正しい位置に移す
#別ブランチのコミットを現在のブランチにも追加する
#$ git cherry-pick
$ git diff 変更前のコミット識別子 変更後のコミット識別子(差分を確認する)
#$ git addのまえにgit diffのみ記述すると、編集した個所と直前のコミットの差分が確認できる。
$ git grep(ファイルから文字列を検索する)
#リポジトリ内の実を検索するため、普通のgrepコマンドよりも検索速度が早い
#リポジトリ内で追跡しているファイルのみを対象に検索ができる
#過去の時点でのファイルの状態を指定して(HEAD^などで)検索できる
#$ git config -global grep.lineNumber trueと設定することで、git grepの検索時に、行数が表示される
#コミット識別子などを指定して、その時点でのファイルの状態を確認できる
$ git blame src/app/app.component.ts(誰のコミットで各行が変更されたのかを表示する)
```

モード名|HEADの位置|インデックス|ワークツリー
---|---|---|---
soft|変更する|変更しない|変更しない
mixed|変更する|変更する|変更しない
hard|変更する|変更する|変更する


# 基本コマンド
```
$ pwd(カレントディレクトリを表示する)
$ ls(カレントディレクトリ直下にあるものを表示する)
#Windowsコマンドプロンプトの場合は「dir」
$ cd(ディレクトリを移動する)
$ mkdir(カレントディレクトリにフォルダを作成する)
$echo(文字列を操作するコマンド)
```
